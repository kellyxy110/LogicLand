# LogicLand — UI Guidelines

## Feel
Modern, warm, magical, professional — **not childish**. We evoke wonder with
restraint, not clutter.

## Rules
- **Minimal reading** on child screens: icons + short phrases.
- **Big, obvious targets** (≥ 44px), generous spacing, rounded corners (xl–3xl).
- **One primary action** per screen for children.
- **Reassure on error** ("Let's try again!"), celebrate on success.
- **Every view** ships loading skeletons, empty states, and an error boundary.
- **Consistency** via `packages/ui` + Tailwind tokens; no one-off colors.
- **Accessibility first:** WCAG AA, visible focus, dark mode, reduced-motion respected.
- **Responsive:** mobile/tablet-first for students; comfortable desktop for teachers.

## Do / Don't
- Do: soft gradients, friendly illustration, purposeful motion.
- Don't: dense text, tiny controls, harsh error dialogs, cartoonish overload.
